INSTRUCTIONS FOR CARLA: GALICIAN EVALUATION
===========================================
Date written: 2026-05-13
Film: ivan-vas (the only film with Russian→Galician translations so far)
Translation model: gpt-5.2
Evaluation model: gpt-5.4-mini

All commands below should be run from the repo root:
  cd /mnt/kesha/armen-subtitulos

You will need a valid OPENAI_API_KEY environment variable set.

STEP 1 — MOVE ANY EXISTING RESULTS, THEN RUN THE EVALUATION
------------------------------------------------------------
Before running the evaluation, follow the instructions in
1-instructions_dir_structure.txt to check whether
existing results need to be migrated to the new directory structure.

Now run the evaluation (the script will add to existing results
in mqm-memes/ without overwriting them):

  python3 code/evaluate_mqm_parallel.py ivan-vas Russian Galician gpt-5.2 gpt-5.4-mini 4 films/prompts/eval/mqm-memes.txt

This will write output into:
  films/output/eval/llm-eval/ivan-vas/Russian-Galician/gpt-5.2-by-gpt-5.4-mini/mqm-memes/


STEP 2 — AGGREGATE THE RESULTS
--------------------------------
aggregate_mqm.py collects all the individual per-subtitle scores produced
in Step 1 and computes summary statistics (mean, std, etc.) for each
translation method, writing them to aggregated_summary.json and
method_comparison.json in the same output directory.

  python3 code/aggregate_mqm.py ivan-vas gpt-5.2-by-gpt-5.4-mini/mqm-memes Russian Galician

Important: the second argument must include the prompt subdirectory
(mqm-memes). If you pass just gpt-5.2-by-gpt-5.4-mini, the script
will treat any subdirectories (prompt variants) as if they were method names
and produce wrong results.


STEP 3 — CHECK VARIANCE
------------------------
variance.py estimates how many translation runs and evaluation runs are
needed to reliably distinguish between methods at a given sensitivity
threshold, based on the variance observed in the current results.

  python3 code/variance.py ivan-vas gpt-5.2-by-gpt-5.4-mini/mqm-memes 0.1 Russian Galician

Same note applies: include the mqm-memes subdirectory in the argument.

The delta (0.1) is the sensitivity threshold — we want to be able to
detect a difference of 0.1 major-equivalent issues per unit between methods.

Read the output for each method:
  - "N translations needed: X" — if X > current number of runs,
    you need more translations (see Step 4).
  - "N eval runs needed: X" — if X > 4, rerun Step 1 with a higher
    eval_runs argument for that method.
  - If everything is within bounds, skip to Step 5.

Look at the actual sensitivity: if it is already close to 0.05, or even 0.08, then rerun this step with 0.05 (or 0.08) and see how many more translations would be needed to reach this better sensitivity. If it is only a few (less than 5-6), then it is worth doing.

STEP 4 — PRODUCE MORE TRANSLATIONS (if needed)
-----------------------------------------------
For each method where variance.py says more translations are needed,
run translate.py. Temperature for gpt-5.2 is 0.8. Example for "zero":

  python3 code/translate.py ivan-vas zero gpt-5.2 0.8 3 Russian Galician

The zero method does not need any extra arguments. For the other methods,
all require --prompt and --summary, except list-analysis which uses
--unit_list instead of --summary:
  - characters:    --prompt films/prompts/characters.txt
                   --summary films/data/ivan-vas/summaries/characters.txt
  - narratives:    --prompt films/prompts/narratives.txt
                   --summary films/data/ivan-vas/summaries/narratives.txt
  - list-analysis: --prompt films/prompts/list-analysis.txt
                   --unit_list films/data/ivan-vas/meme-list-analysis.json
  - noise:         --prompt films/prompts/characters.txt
                   --summary <summary/character list from a DIFFERENT film>
                   (the point is to feed the model misleading context)

After producing more translations, rerun Steps 1–3 for the affected
methods until you are satisifed with the results. If a particular method seems to require an unreasable amount of data (e.g. 30 translation runs or something like that), do not try to reach target sensitivity for that method (see narratives example for pokrov-gate eng).
